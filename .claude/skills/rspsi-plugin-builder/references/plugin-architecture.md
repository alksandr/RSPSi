# RSPSi Plugin Architecture Reference

## Plugin System Overview

RSPSi uses Java ServiceLoader to discover and load plugins at runtime. Plugins replace default cache loaders with revision-specific implementations.

## Plugin Lifecycle

1. **Discovery**: `ClientPluginLoader` scans `Editor/plugins/active/` for JARs
2. **Loading**: URLClassLoader loads plugin classes, ServiceLoader finds implementations
3. **Initialization**: `plugin.initializePlugin()` — registers loader singletons
4. **Cache Load**: Client loads cache structures (indices, archives)
5. **Game Loaded**: `plugin.onGameLoaded(client)` — initializes loaders with cache data
6. **Resource Delivery**: `plugin.onResourceDelivered(response)` — handles lazy-loaded resources

## Plugin Interfaces

### ClientPlugin (com.rspsi.plugins.core.ClientPlugin)
```java
public interface ClientPlugin {
    void initializePlugin();                              // Register loaders
    void onGameLoaded(Client client) throws Exception;    // Init with cache data
    default void onResourceDelivered(ResourceResponse r) {} // Handle lazy loads
}
```

### ApplicationPlugin (com.rspsi.plugins.ui.ApplicationPlugin)
```java
public interface ApplicationPlugin {
    void initialize(MainWindow window);  // Extend editor UI
}
```

## Loader Registration Pattern

Each loader type has a static `instance` field. Plugins replace these in `initializePlugin()`:

```java
ObjectDefinitionLoader.instance = new MyObjectDefinitionLoader();
FloorDefinitionLoader.instance = new MyFloorDefinitionLoader();
MapIndexLoader.instance = new MyMapIndexLoader();
TextureLoader.instance = new MyTextureLoader();
AnimationDefinitionLoader.instance = new MyAnimationDefinitionLoader();
FrameLoader.instance = new MyFrameLoader();
FrameBaseLoader.instance = new MyFrameBaseLoader();
GraphicLoader.instance = new MyGraphicLoader();
VariableBitLoader.instance = new MyVarbitLoader();
RSAreaLoader.instance = new MyRSAreaLoader();
```

## Build Configuration

All plugins share `Plugins/build.gradle`:
- Uses `com.github.harbby.gradle.serviceloader` plugin v1.1.9
- Registers: `com.rspsi.plugins.core.ClientPlugin`, `com.rspsi.plugins.ui.ApplicationPlugin`
- Dependency: `implementation project(':Editor')`
- Task `buildAndMove`: compiles + generates ServiceLoader metadata + creates JAR + copies to `Editor/plugins/inactive/`
- Java 21 (inherited from root project)

## File Structure for a Plugin

```
Plugins/{PluginName}/
├── src/main/java/com/rspsi/plugin/
│   ├── OSRSPlugin.java                    # implements ClientPlugin
│   └── loader/
│       ├── ObjectDefinitionLoaderOSRS.java
```

## Cache Index Constants

Standard OSRS cache layout (accessed via `CacheFileType` enum):
- Index 0: ANIMATION (frame groups) — `CacheFileType.ANIMATION`
- Index 1: SKELETON (frame bases) — `CacheFileType.SKELETON`
- Index 2: CONFIG (definition archives) — `CacheFileType.CONFIG`
- Index 3: INTERFACES
- Index 4: SOUNDEFFECTS
- Index 5: MAP (terrain + object placements) — `CacheFileType.MAP`
- Index 6: MUSIC
- Index 7: MODEL — `CacheFileType.MODEL`
- Index 8: SPRITE — `CacheFileType.SPRITE`
- Index 9: TEXTURE — `CacheFileType.TEXTURE`

Config archive IDs (within index 2):
- 1: UNDERLAY (floor underlays)
- 4: OVERLAY (floor overlays)
- 6: OBJECT (game objects)
- 9: NPC
- 10: ITEM
- 12: SEQUENCE (animations)
- 13: SPOTANIM (graphics/spot animations)
- 14: VARBIT (variable bits)
- 35: AREA (RSArea / minimap functions)

## RSPSi Buffer API

Common read methods on `com.jagex.io.Buffer`:
- `readByte()` / `readUByte()` — signed/unsigned byte
- `readShort()` / `readUShort()` — signed/unsigned 16-bit
- `readInt()` — 32-bit integer
- `readUTriByte()` — unsigned 24-bit integer
- `readOSRSString()` — null-terminated CP1252 string (byte 0x00, use for OSRS cache data)
- `readString()` — newline-terminated string (byte 0x0A, for 317-era formats only)
- `readSmart()` — variable-length smart int
- `readShort2()` — big-endian short (used in frame data)
- `skip(n)` — skip n bytes

For ByteBuffer-based loaders (used by FloorDefinitionLoaderOSRS, RSAreaLoaderOSRS):
- `buffer.get()` — byte
- `buffer.getShort()` — short
- `buffer.getInt()` — int
- Manual unsigned: `buffer.get() & 0xFF`, `buffer.getShort() & 0xFFFF`
- `ByteBufferUtils.getSmartInt(buffer)` — smart int
- `ByteBufferUtils.getOSRSString(buffer)` — null-terminated string
- `ByteBufferUtils.getMedium(buffer)` — 24-bit int

## Definition Class Key Fields

### ObjectDefinition
Core fields: id, name, modelIds[], modelTypes[], width, length, animation,
interactive, solid, impenetrable, contouredGround, delayShading, occludes,
castsShadow, ambientLighting (byte), lightDiffusion (byte),
decorDisplacement, scaleX/Y/Z, translateX/Y/Z, interactions[] (5 elements for 218),
originalColours[], replacementColours[], retextureToFind[], textureToReplace[],
morphisms[] (with varbit, varp fields), areaId, mapscene, hollow,
supportItems, surroundings, inverted, obstructsGround, category, randomizeAnimStart

### Floor
Core fields: rgb, anotherRgb, texture, shadowed (boolean),
hue, saturation, luminance, anotherHue, anotherSaturation, anotherLuminance,
weightedHue, chroma, colour

### Animation
Core fields: frameCount, primaryFrames[], secondaryFrames[], durations[],
loopOffset, interleaveOrder[], stretches, priority, playerOffhand, playerMainhand,
maximumLoops, animatingPrecedence, walkingPrecedence, replayMode

### Graphic
Core fields: id, model, animationId, animation (Animation object),
breadthScale, depthScale, orientation, ambience, modelShadow,
originalColours[6], replacementColours[6]

### VariableBits
Core fields: setting, low, high
